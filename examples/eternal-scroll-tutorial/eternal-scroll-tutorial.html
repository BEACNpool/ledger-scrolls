The Eternal Scroll

How to read & write immutable media on Cardano

You are reading a document that lives on the Cardano blockchain.
No server hosts it. No one can edit it. It cannot be taken down. Ever.

If you can see this page, your viewer just performed a small miracle: it asked the Cardano blockchain for a handful of transactions, reassembled their bytes in order, decompressed them, computed a SHA-256 hash, and compared it against a fingerprint locked forever in the ledger. Only when the fingerprints matched did it show you a single word.

That is a Ledger Scroll — and this tutorial will teach you to read them, verify them with your own hands, and write your own.

What Is a Ledger Scroll?

A Ledger Scroll is any file — text, image, HTML, PDF, audio, video — stored byte-for-byte inside Cardano transactions, together with a cryptographic fingerprint (SHA-256) that lets anyone verify a perfect reconstruction. Not a link to a file. Not a hash of a file kept somewhere else. The file itself, on the chain that thousands of independent nodes replicate around the world.

  • Permissionless — no account, no gatekeeper, no platform. Anyone with a few ADA can publish.
  • Immutable — once written, the bytes can never be altered by anyone, including the author.
  • Verifiable — readers don't trust; they check. The hash either matches or it doesn't.
  • Unkillable — as long as one copy of the Cardano chain exists anywhere, every scroll survives.

How Reading Works

Everything starts from a pointer — a single transaction id, small enough to share as a QR code. It leads to a manifest: a tiny record locked inside an unspendable Cardano UTxO, which means every Cardano node carries it in live state, forever, retrievable with one query. The manifest declares what the file is, lists the exact transactions holding its pages, and commits to the SHA-256 of the finished file. The reader fetches those pages, reassembles, and proves the result.

The golden rule: a viewer never shows you bytes it could not verify. “Trust” is replaced by arithmetic.

Verify This Scroll With Your Own Hands

Don't take this page's word for anything — that's the point. With free, keyless tools (the Koios public API and the open-source viewer), you can rebuild this document yourself:

git clone https://github.com/BEACNpool/ledger-scrolls
cd ledger-scrolls/koios-viewer

# rebuild this very document from the chain:
python3 -m lsview reconstruct-chain --txin <manifest-txin> --out scroll.html

sha256sum scroll.html   # compare with the manifest's fingerprint

The viewer prints the manifest's declared hash and verifies every page against it. If a single bit anywhere were wrong, the reconstruction would fail loudly. It never has — because the chain cannot change.

No tooling at all? Verify with two web requests

You don't even need our software. The entire protocol is two queries against any Cardano indexer. First, fetch the manifest datum (one POST to Koios — free, no API key):

curl -X POST https://api.koios.rest/api/v1/utxo_info \
  -H 'Content-Type: application/json' \
  -d '{"_utxo_refs":["<manifest-txhash>#0"], "_extended":true}'
# the inline_datum bytes decode to: content type, codec, file size,
# two SHA-256 fingerprints, and the ordered list of page transactions

Then fetch the pages it names:

curl -X POST https://api.koios.rest/api/v1/tx_metadata \
  -H 'Content-Type: application/json' \
  -d '{"_tx_hashes":["<page-tx-1>","<page-tx-2>"]}'
# label 22025 of each tx holds the payload segments; concatenate them
# in manifest order, gunzip, sha256sum — done

Concatenate, decompress, hash. Thirty lines of any language you like — the repository ships test vectors so your thirty lines and ours can prove they agree. A protocol simple enough to re-implement from its spec in an afternoon is a protocol that survives its creators.

Anatomy of a manifest

The manifest is 8 fields of CBOR locked in the datum — small enough to read aloud:

Constr 0 [
  version        = 2
  contentType    = "text/html"
  codec          = "gzip"            # or "none"
  sizeDecoded    = <bytes of the original file>
  sha256Decoded  = <fingerprint of the original file>
  sha256Encoded  = <fingerprint of the compressed stream>
  pageTxHashes   = [ tx₁, tx₂, … ]   # the pages, in order
  next           = none              # or a link to a continuation manifest
]

That's the whole secret. Every field exists to answer one question: “are these exactly the author's bytes?”

Write Your Own — Step by Step

1 · Prepare your file

Any file works; small files cost less. Optimize images, strip private EXIF metadata, make HTML self-contained (inline styles, embedded images, no scripts — viewers run scrolls in a strict sandbox). Then:

python3 tools/lschain/prepare.py mywork.html --content-type text/html
#  -> gzips deterministically, hashes everything, splits into pages,
#     prints the page count and an honest cost estimate BEFORE you spend

2 · Mint

tools/lschain/mint.sh lschain-work payment.skey payment.addr
#  -> submits page transactions (chained, no waiting), then locks the
#     manifest at the always-fail address; writes receipts.json

Each page is an ordinary transaction fee (~0.84 ADA per 12 KB page at 2026 parameters). The only ADA locked forever is the manifest's minimum deposit (~2 ADA). A 100 KB illustrated essay: roughly 9 pages ≈ 8 ADA. This entire tutorial: a few ADA. Eternity is cheaper than a coffee subscription.

3 · Verify, then announce

python3 -m lsview reconstruct-chain --txin <your-manifest>#0 --out check.html
sha256sum check.html mywork.html   # identical, or you don't announce

Keep receipts.json. Share your manifest pointer. You have published something no outage, takedown, bankruptcy, or regime change can ever unpublish.

The Five Rules of the Scroll

  1. Permanent means permanent. No edit, no delete. Never inscribe secrets, private data, or words you may regret. The chain forgives nothing.
  2. Rehearse on testnet. Same bytes, free mistakes.
  3. Hash before, verify after. Record your file's SHA-256 before minting; reconstruct from chain and compare before telling the world.
  4. Keep receipts. Transaction hashes are how civilization finds your scroll again.
  5. Self-contained files only. Anything that phones home — external images, fonts, scripts — is a future broken promise. Inline everything.

Questions People Actually Ask

QuestionAnswer
Who controls my scroll after minting? No one — not you, not BEACN, not Cardano's founders. The manifest sits at an address whose spending script always fails; the pages are ordinary confirmed transactions. There is no admin key anywhere in the design.
What if BEACN disappears? Nothing happens to any scroll. The viewer is open source, the spec is public, the data is on a public chain, and the protocol can be re-implemented from the spec alone. BEACN is a librarian, not a landlord.
Do I get an NFT? No — and that's a feature. Older scrolls used NFTs to carry pages, which meant tokens that could be moved or lost. LS-CHAIN pages are plain transactions: nothing to custody, nothing to misplace, nothing to rug.
Is it really forever? As long as any copy of the Cardano chain exists. Transaction history is what every node syncs from; the manifest additionally lives in the active UTxO set that every node holds in full. Your scroll's survival is bound to the survival of the ledger itself.
Can someone else claim they wrote it? The chain timestamps your scroll in a specific block, signed by your keys. First inscription wins every argument thereafter.
What does it cost? Network fees only, plus ~2 ADA locked with the manifest. This tutorial cost a few ADA. The chain shows you the exact price before you sign — honesty is part of the protocol.

Why This Matters

Libraries burn. Servers go dark. Companies fold, links rot, platforms moderate, archives get edited. Every promise of digital permanence so far has depended on someone continuing to pay, to host, to care.

A Ledger Scroll depends on none of that. It rides inside the same replicated ledger that secures billions in value, copied onto thousands of machines whose operators have their own reasons to keep every byte intact. Your words, your art, your evidence — preserved by physics and incentive, not by goodwill.
